Vorbereitungen für N.A.S.E.

  • CVS-Root-Directory setzen:

    bash: export CVSROOT=/usr/ax1303/neuroadm/IDLCVS
    tcsh: setenv CVSROOT /usr/ax1303/neuroadm/IDLCVS

Grundlegende Aktionen

Erstellen einer Arbeitskopie

  • In ein beliegibes Verzeichnis wechseln, unter dem die Directories und Files angelegt werden sollen.

      • cvs checkout nase

Updaten einer Arbeitskopie

  • Hat man bereits eine Arbeitskopie könnte es ja sein, daß jemand anderes Teile des Projekts verändert hat. Um die vorhandene Version zu aktualisieren, ohne eigene Veränderungen zu überschreiben benutzt man:

      • cvs update nase

Editieren eines Files

  • Als erstes sollte man überprüfen, ob nicht bereits jemand am gleichen File herumwurschtelt:

      • cvs editors

  • Wenn nicht, kann man das File zum editieren markieren:

      • cvs edit <file>

  • Die vorher schreibgeschützte Version ist nun schreibbar. Ist man fertig und stellt fest, daß alle Veränderungen Mist sind und man lieber den ganzen Quatsch vergessen möchte, benutzt man am besten

      • cvs unedit <file>

  • Fktioniert alles prima, kann man die Änderungen für alle sichtbar machen:

      • cvs commit <file>

Hinzufügen eines neuen Files

  • Möchte man ein neues File hinzufügen, dann muss man eingeben:

      • cvs add <file>

  • Damit alle Änderungen wirksam werden, gibt man dann noch ein abschließendes

      • cvs commit <file>

Löschen der Arbeitskopie

  • Hat man keine Lust mehr, was zu arbeiten, kann man die Arbeitskopie löschen:

      • cvs release -d nase

Tags & Releases

Erstellen eines Releases

Da die Funktionalität und die Aufrufkonventionen vieler Routinen mit der Zeit verändert wird, kann es vorkommen, daß alte Simulationen mit den neuen Routinen nicht mehr zusammenpassen. Möchte (oder kann) man die Simulationen nicht anpassen, so braucht man die alten Routinen. Kein Problem, aber welche Revisionsnummern hatten die ?? CVS bietet die Möglichkeit symbolische Revisionsnamen zu vergeben. Dies funktioniert für einzelne Files:

    cvs tag <release-tag> simu/delayweigh.pro

und auch rekursiv für alle Files in und unter dem aktuellen Verzeichnis:

    cvs tag <release-tag> .

In den Tag-Namen sind die Symbole '$ , . : ; @' nicht zugelassen.

Anzeigen von Tags

Die existierenden Tags, sowie allemeine Informationen zu einem CVS-File erhält man mit:

    cvs status -v delayweigh.pro

Auschecken eines bestimmten Releases

    cvs checkout -r release-5-9-97 nase

Dieses Release gibt's wirklich, kann mal ausprobiert werden.

Eine andere Möglichkeit besteht darin alle Files auszuchecken, den zu einem bestimmten Datum (i.a. dem der Simulation) aktuell waren:

    cvs checkout -D <Datum-String> nase

Als Datum-String sind Sachen wie

    '500000 seconds ago', 'last year', 'yesterday', 'January 23, 1987 10:05pm'

erlaubt.
